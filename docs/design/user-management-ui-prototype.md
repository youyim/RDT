## 3. 交互说明 (Interaction / Behaviors)

### 状态变化

- **Loading**: 表格区域展示 5 行 Skeleton。
- **Empty**: 列表为空时，展示 `EmptyState` 组件（插画 + "暂无用户" + "新建"按钮）。
- **Hover**: 鼠标悬停表格行时，背景色加深 (`bg-hover`)，显现操作按钮。

### 关键流转

1.  **点击 [Add User]**:

    - 弹出模态框 (UserModal)，Title="新增用户"。
    - 表单重置为空。
    - 点击 Confirm -> API `POST /api/v1/users` -> Toast "创建成功" -> Close Modal -> 刷新列表。

2.  **点击 [Edit]**:

    - 获取当前行数据，填充模态框，Title="编辑用户"。
    - 点击 Confirm -> API `PUT /api/v1/users/{id}` -> Toast "更新成功" -> Close Modal -> 刷新列表。

3.  **点击 [Reset Password]**:

    - 弹出 Confirm Dialog: "确定要重置该用户的密码吗？"
    - Confirm -> API `POST /api/v1/users/{id}/reset-password` -> Toast "重置邮件已发送"。

4.  **点击 [Delete]**:
    - 弹出 Delete Dialog (Danger): "确定要归档该用户吗？此操作不可逆。"
    - Confirm -> API `DELETE /api/v1/users/{id}` -> Toast "用户已归档" -> 刷新列表。

## 4. 响应式策略 (Responsive Strategy)

| 断点                | 布局调整                                                                                             |

| **Desktop**         | 标准表格布局，操作列固定在右侧。                                                                     |
